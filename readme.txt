This is a little project with a very simple pattern that allows to maintains data updated based on the Q_PROPERTY.
The user must write the get/set logic and also the UI.
The user of the pattern must respect just  a name convention and the framework connect all the signal/slot in the right way.
An utility to serialize data with a doom structure is provided, it's very simple to copy/paste data and also to store/import to/from an xml file 

More detail in the doc directory.
See test1 as example, two instance are created and it's possible copy&paste data between them also import/export file.
